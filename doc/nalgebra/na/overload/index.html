Traits to work around the language limitations related to operator overloading.

The trait names are formed by:

  • a type name (eg. Vec1, Vec2, Mat3, Mat4, etc.).
  • the name of a binary operation (eg. Mul, Div, Add, Sub, etc.).
  • the word Rhs.

When implemented by the type T, the trait makes it possible to overload the binary operator between T and the type name given by the trait.

Examples:

  • Vec3MulRhs will allow the overload of the * operator between the implementor type and Vec3. The Vec3 being the first argument of the multiplication.
  • Mat4DivRhs will allow the overload of the / operator between the implementor type and Mat4. The Mat4 being the first argument of the division.
